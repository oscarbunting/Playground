Differences between let and const

Const, which stands for constant, is used to declare variables that aren't supposed to change, for example:
const englandCapital = "London";

To make it extra clear that a variable is constant and shouldn't chnage, we can name it in all caps, for example:
const BIRTHDAY = "23/08/03"

We can't use camelCase if we use the uppercase standard, so we use snake_case instead when we make the variable name all caps. If we try and update a constant variable, we will throw an error. On the other hand, variables that are declared using let can be updated by using the assign = operator. For example we could create a variable like this:
let oscarAge = 20
And then update the variable at a later time like this:
oscarAge = 21

We'll need to use both let and const, and in theory we could use them interchangeably. However, we should try to use const by default

Comparing Numbers

To check if a number is less than another, we use the less-than operator, <.
For example, here we want to see if 10 is less than 100:

console.log(10 < 100)

Output/Console:

true

If the number on the left is less than the number on the right, we will get true in return, and if the number on the left is bigger than the number on the right, we will get false.
Similarly, if we want to check if one number is bigger than another, we can use the greater-than operator, >.
For example:

console.log(100 > 10)

Output/Console:

true

To check if a number is less than equal or equal to another rnumber, we use the less-than-or-equal-to operator, <=.
If the number on the lest is less than or equal to the number on the right, (for example 11 <= 11), the output will be true.

If we want to check if a number is greater than or equal to another, we can use the greater-than-or-equal-to operartor, >=.

To store the result of a comparison in a variable, we use the = sign. This is the assignment operator.

We can also use a comparison operator to compare a variable with another variable, like in the min <= max example below.

const min = 5;
const max = 10;
const result = min <= max;
console.log(result);

Console/Output:

true

Comparing Strings

We can also use comparisons to check if a sting is equal, (or not equal) to another number.
To check if a string is equal to another we can use the strict equality operator, ===.

If the string on the left is equal to the string on the right as in "oscar" === "oscar", the result is true.

If the string on the left isn't equal to the one on the right, as in "oscar" === "bunting", the result is false. We can also compare variables that store strings with each other, like in fruit1 === fruit2.

const fruit1 = "apple";
const fruit2 = "orange";
console.log(fruit1 === fruit2);

Console/Output:

false

To check if a string isn't equal to another, we can use the strict inequality operator, !==.
If the string on the left isn't equal to the string on the right, as in "apple" !== "orange", the result is true.

Combining values with +

+ is the joining or adding sign. When joining two variables, we can get different results depending on their types.
Strings and numbers will act differently when joined or added to one another. For example:

console.log("10" + "10"); would output 1010 because they are strings, not numeric values.
On the other hand, console.log(10 + 10); would output 20, because this time we used a numeric value.

If we try to combine numbers and strings, the number will automagically be transformed into a string, for example: console.log(10 + "10"); would output 1010 because the numeric value was turned into a string.

Adding a number to a string also works when we use a variable like score that stores a number, instead of a numeric value. For example:

const score = 99;
const result = "Your score: " + score ;
console.log(result);

Output/Console:

Your score: 99

 We can also combine strings and booleans together, or (just like with numbers and strings) a variable that has a value of a boolean will work too. Also just like with strings and numbers, when we combine a boolean and a string, the boolean is converted to a string.

For example, console.log("You Passed: " + true); would output You Passed: true.

Introduction to Logical Operators

With logical operators, we can connect two or more conditions to decide a result.
The and operator, && will only return true if all the conditions are also true. For example:

let isBatteryOn = true;
let isSwitchOn = true;
console.log(isBatteryOn && isSwitchOn);

Console/Output:

true

In the above example, we want to check that both the isBatteryOn variable and the isSwitchOn variable are true before out putting true, so we used the && operator to make sure they're both true. If one of the values was false the final output would also be false
We can also store the result of a logical exprerssion in a variable, for example let result = isBatteryOn && isSwitchOn; to be called back later.

We can also use the and operator to check if other operators are true, and only return true if the other rules have returned true. In the example below, we want to check if the cost is less than the selling price, and also make sure that the profit is above zero. Only if both of these conditions are met should true be returned. lets see how we can make this happen:

let cost = 50;
let sellPrice = 60;
const profit = sellPrice - cost;
const result = cost < sellPrice && profit > 0;
console.log(result);

Output/Console:

true

This is a bit tricky so we'll do one more example. In this next example we only want to output true if we can confirm that the user is an admin. In this case, admins must be signed in, have a level of 20 or more and also have the isAdmin value be true. We can only grant admin access if all of these are true.
let isAdmin = true;
let isLoggedIn = true;
let acessLevel = 30;
let adminAcess = isAdmin && isLoggedIn && acessLevel >= 20;
console.log(adminAcess);

Console/Output:

true

The OR logical operator ||

The and operator requires all of the conditions to return true, but what if we only want one condition to be true? Introducing the or operator, if all the conditions are false then it will return false, however if only one is true it will output true.

We already know that the NOT operator (!) negates a boolean value. That means that it returns true is a condition is false and vice versa.
We can also use ! to negate logical values as well. To do that, we place the logical expression between parentheses, like in the example below:

let cost = 50;
let sellPrice = 60;
const profit = sellPrice - cost;
const result = !(cost < sellPrice && profit > 0);
console.log(result);

Output/Console:

false

The above example should output true, however because we used the not operator and added parentheses, it displays the opposite.

Making Decisions

Smart programs use booleans to decide whether to run lines of code or skip them.
How can we write code that adapts to different situations?
By uisng the if statement. We code the if keywork for an if statement. For example:

if (true) {
console.log("Hello World");
}

Console / Output:

Hello World

The if statement only runs code if the boolean its relying on is true.
It's like telling the computer if something is true then do this.

Values like true are called conditions. Statements relying on conditions are called conditionals.

Conditions decide if the code runs or gets skipped. They come in between parentheses.
if statements don't decide on skipping or running the entire codebasde, they only make deciions about a piece of code. The piece of code that is either run or skipped is called a code block. We write it between curly braces { }. For example:

if (true) {
console.log("I'm a code block!");
console.log("I'm a second line in a code block");
}

Output/console:

I'm a code block!
I'm a second line in a code block

A code block can be more than a one-liner, we can use as many lines as we'd like.

Instead of uisng the boolean true, we can save it in a variable and use that as a conditon.
For example in the below snippet, we're storing the boolean in the variable greet and using that to see if we should run our code block.

const greet = true;
if (greet) {
console.log("Hello!");
}

Output/console:

Hello!

Only if the greet variable contains the value true will our code block run.

Using Conditions

To make smarter programs we need if statements that rely on smarter conditions. In the example below we're creating a quiz that need to check if the answer is wrong or right.
We can acomplish this with a combintions of if statements and logical operators. Lets take a look:

let answer = "Picasso";
if (answer === "Picasso") {
console.log(answer + " is correct!");
}
if (answer !== "Picasso") {
console.log(answer + " is wrong!");
}

Output / Console:

If correct: Picasso is correct!
If incorrect: (answer) is wrong!

if statements work with all the comparison operators we've explored so far, we can also use === to compare boolean values within an if statement.
